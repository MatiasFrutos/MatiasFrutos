### 🧠 ¿Qué es API Watchtower?

**API Watchtower** es una aplicación de escritorio pensada para developers que necesitan probar endpoints, validar respuestas, medir tiempos, revisar errores y mantener un historial local sin depender de una plataforma pesada.

La idea principal es simple:

```txt
Open app → Create project → Add endpoint → Send request → Review response → Check history
```

No requiere cuenta.  
No depende de la nube.  
No necesita configuración compleja.  
Todo queda local.

---

### 🚀 Funciones principales

| Módulo | Descripción |
|---|---|
| 📁 Projects | Crear y organizar proyectos API con nombre, base URL y descripción. |
| 🔗 Endpoints | Registrar endpoints con método HTTP, URL, headers y body. |
| 🧪 Tester | Ejecutar requests reales desde la app desktop. |
| 🕘 History | Guardar y revisar historial local de requests ejecutados. |
| 📊 Dashboard | Ver resumen de proyectos, endpoints, requests y tiempos promedio. |
| 📈 Reports | Generar reportes locales de disponibilidad, errores y rendimiento. |
| ⚙️ Settings | Configurar parámetros generales de la herramienta. |
| 💾 Local Storage | Persistencia simple en archivos JSON locales. |

## 🧾 Breve descripción de los proyectos

### 🛡️ API Watchtower

**API Watchtower** es una aplicación desktop local-first para probar APIs de forma simple, rápida y ordenada.

Permite crear proyectos, registrar endpoints, ejecutar requests reales, medir tiempos de respuesta, guardar historial local y generar reportes básicos. Está pensada como una alternativa liviana para desarrolladores que necesitan validar APIs sin depender de cuentas externas, nubes o herramientas demasiado pesadas.

Es el proyecto que más destaco actualmente porque combina frontend, backend, Electron, arquitectura modular, persistencia local y una problemática real del desarrollo diario.

---

### 🧠 StackForge

**StackForge** es un IDE visual e interactivo para programar en **Forth**, un lenguaje basado en pila.

El objetivo del proyecto es hacer más fácil entender cómo funciona internamente la ejecución de un programa Forth. Mientras el usuario escribe y ejecuta código, la herramienta permite visualizar la pila de datos, analizar cada operación, usar un REPL integrado, consultar el diccionario de palabras y revisar la ejecución paso a paso.

Es un proyecto con enfoque educativo, ideal para explorar lenguajes stack-based, lógica interna de ejecución y herramientas visuales para aprendizaje técnico.

---

### 🧭 SheetPilot

**SheetPilot** es una herramienta web para limpiar, ordenar y transformar archivos de datos desordenados.

Está pensada para resolver un problema muy común en entornos administrativos y técnicos: archivos CSV, TXT o Excel que llegan mal formateados, con datos mezclados o separados incorrectamente. La herramienta detecta separadores, interpreta filas, propone columnas, permite editar la estructura y exportar el resultado en un formato más limpio y útil.

Su foco está en productividad, limpieza de datos y reducción de tareas manuales repetitivas.

---

### 📁 ZERNYX File Organizer

**ZERNYX File Organizer** es una aplicación de escritorio creada en Python para ordenar archivos automáticamente por tipo.

Permite seleccionar una carpeta, analizar su contenido, previsualizar el destino de cada archivo, organizar documentos, imágenes, planillas, videos, código, instaladores y otros formatos. También permite incluir o no subcarpetas, evitar sobrescrituras, crear carpetas organizadas por fecha, ver historial y deshacer el último movimiento.

Es una herramienta práctica para escritorios, descargas, carpetas de clientes, archivos administrativos y entornos PYME donde la información suele acumularse sin orden.

---

### 🧠 SQLMind Local

**SQLMind Local** es un asistente SQL local-first para generar, explicar, optimizar y documentar consultas SQL.

Permite escribir una necesidad en lenguaje natural y obtener una consulta SQL, pegar una query existente para entender qué hace, recibir sugerencias de optimización o generar estructuras `CREATE TABLE` desde una descripción. Funciona con un proveedor fake local para pruebas y puede integrarse con modelos locales mediante Ollama.

El proyecto está pensado para desarrolladores, estudiantes, analistas y equipos técnicos que trabajan con bases de datos y quieren una herramienta simple, local y extensible.

---

### ⚡ DevProject Starter Generator

**DevProject Starter Generator** es una herramienta CLI para generar estructuras base de proyectos desde la terminal.

Permite crear starters frontend, backend, fullstack y módulos ERP/CRM con una estructura limpia y reutilizable. El objetivo es evitar repetir siempre el mismo setup manual de carpetas, archivos, configuración inicial, servicios, rutas y componentes base.

Es una herramienta orientada a productividad developer, especialmente útil para iniciar proyectos rápidos, prototipos, paneles administrativos o módulos de sistemas internos.

## 🧭 Forma de trabajo

Me gusta trabajar con una mentalidad práctica: primero entender el problema, después construir una versión funcional y luego mejorarla.

Mi enfoque suele ser:

- Analizar el problema antes de escribir código
- Priorizar una base funcional
- Mantener una estructura clara
- Evitar complicar de más lo que puede resolverse simple
- Documentar lo necesario
- Automatizar cuando realmente aporta valor
- Mejorar con cada iteración

No todo tiene que ser perfecto desde el día uno. Pero sí tiene que avanzar, funcionar y dejar una base ordenada para seguir creciendo.
